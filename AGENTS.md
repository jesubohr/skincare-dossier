# Codex usage guidelines

## DO NOT DO

- Do not change the color palette, typography, spacing, or layout system unless explicitly authorized
- Do not overwrite custom CSS or Tailwind classes unless intended
- Do not modify core authentication, data handling, or navigation logic without review

## DO

- Follow the existing design system and styling patterns detailed in `DESIGN.md`
- Maintain the minimalist, professional tone in all copy and UI text

## General Guidelines

- **Consistency is critical**: The app has a very specific aesthetic—preserve it
- **Professional tone**: Keep all language polished and appropriate for a medical-spa setting
- **Simplicity**: Avoid unnecessary complexity in design or functionality
- **Performance**: Optimize for speed and minimal resource usage
- **Accessibility**: Follow accessibility best practices

## File Naming Conventions

- Use kebab-case for filenames: `my-component.tsx`, `styles.css`
- Component files: `ComponentName.tsx` (PascalCase for components)
